## 一、项目概述

### 1.1 核心功能

琳琅乐府是一个桌面音乐管理工具，核心做三件事：

1. **AI 歌词识别** — 用 Whisper AI 自动听写歌曲音频，生成 LRC 时间轴歌词。支持多引擎切换（Groq 云端 / 本地 Whisper / 讯飞 API）
2. **歌词管理** — LRC 可视化编辑、搜索、批量管理
3. **跨设备文件同步** — 通过 LocalSend 协议实现手机到电脑的无线文件传输，配合文件夹差异比对，完成手机音乐库到电脑的汇聚

> **关于「同步」**: 当前版本不是传统意义上的"本地两个文件夹同步"。同步流程是：手机端通过 LocalSend App 将文件发送到电脑 → 软件接收并保存到指定文件夹 → 与现有文件做差异比对。后续计划支持双向传输（电脑也可发送到手机）。

### 1.2 关键决策

| 维度 | 决策 |
|------|------|
| 语言 | **中文为主**，支持英语/日语/韩语 |
| 歌词格式 | LRC 时间轴歌词 (`[mm:ss.xx]`) |
| 识别方案 | Groq Whisper（云端免费）→ 本地 Whisper（离线）→ **讯飞 API（规划中）** |
| 人声分离 | 默认关闭，可选 Demucs |
| 音频格式 | MP3 / FLAC / WAV / AAC / M4A / OGG / OPUS |
| 同步方式 | LocalSend Protocol v2.1（手机→电脑）+ HTTP 文件浏览 + mDNS 发现 |
| AI 集成 | **CLI 全覆盖 + --json 输出**，供 AI Agent 直接操控软件 |
| 应用形态 | 桌面端 (PyQt6) + CLI (11 个命令) + AI Agent 接口 |

## 三、AI Agent 集成设计

### 3.1 设计理念

琳琅乐府的 CLI 不是简单的"命令行替代 GUI"——它是**专门为 AI Agent 设计的操控接口**。

人工操作 GUI 需要点击、选择、输入。但 AI 需要的是：
- 发现：扫描环境、了解当前状态
- 决策：根据状态选择操作
- 执行：通过标准接口执行操作
- 验证：获取结构化反馈确认结果

### 3.2 CLI ↔ AI Agent 映射

```
AI Agent 工作流              CLI 命令
─────────────────────────────────────────
1. 了解环境                   config show --json
   检查引擎/模型/GPU状态       gpu status --json
                              model list --json

2. 发现待处理任务              list --status no-lrc --json
   找到所有无歌词的歌曲

3. 执行识别                   transcribe <file> --json --quiet
   逐个或批量处理

4. 验证结果                   lyrics show <file> --json
                              info <file> --json

5. 搜索/管理                  lyrics search "关键词" --json
                              rename / mark
```

### 3.3 后续：智能 Agent 对接

计划接入一个智能 Agent（如 GPT/Claude/本地 LLM），通过 CLI 接口实现自然语言操控：

```
用户: "帮我把所有周杰伦的歌都识别一下歌词"
  ↓
Agent: list --search "周杰伦" --json
  → 找到 15 首
Agent: 遍历 transcribe 每首
  → 完成 15 首
Agent: "已完成！周杰伦 15 首歌全部识别完毕，其中 2 首可能是纯音乐已自动标记"
```

CLI 的所有 `--json` 输出就是为这个场景准备的——Agent 不需要解析人类可读的表格，直接拿 JSON 做逻辑判断。

### 3.4 扩展性：Provider 热插拔

CLI 的 `config set asr.provider` 配合 ASR Provider 模式，允许 Agent 根据不同场景切换引擎：

| Provider | 命令 | 场景 |
|----------|------|------|
| Groq 云端 | `config set asr.provider groq` | 快速、免费、联网 |
| 本地 Whisper | `config set asr.provider local` | 离线、隐私 |
| **讯飞 API** | `config set asr.provider xunfei` | **规划中**，中文识别最佳 |

## 六、文件同步架构

### 6.1 当前：手机→电脑

```
┌──────────────┐     LocalSend v2.1      ┌──────────────┐
│   手机端      │ ◄─────────────────────► │   电脑端      │
│              │   UDP 多播发现设备        │              │
│  LocalSend   │   HTTPS 加密传输         │  琳琅乐府     │
│    App       │ ───── 文件 ─────→        │  (接收端)     │
│              │                         │              │
│  /sdcard/    │                         │  E:\music\   │
│   Music/     │                         │    音乐/      │
└──────────────┘                         └──────────────┘
```

流程：
1. 电脑端开启 LocalSend 接收 → 生成自签名证书 → UDP 多播广播
2. 手机 LocalSend App 发现设备 → 选择文件 → HTTPS 加密传输
3. 电脑端接收 → 保存到指定文件夹 → 自动刷新歌曲列表

### 6.2 后续：双向 + Agent 控制

规划通过 CLI 和 Agent 实现更智能的同步：

```bash
# CLI 对比差异
python main.py sync compare --dir-a E:/music/ --dir-b /sdcard/Music/ --json

# Agent 根据差异自动决策
# "手机上有 5 首新歌电脑没有，自动接收"
# "电脑有 3 首新歌手机没有，通过 HTTP 服务让手机下载"
```

## 七、已知问题 & 卡壳点

### 🔴 阻塞：模型下载 (P0)
- **问题**: Whisper 模型文件只能从 OpenAI CDN / HuggingFace 下载，国内全被 GFW RST
- **当前方案**: 模型放在 GitHub Releases (`echovault-models`)，软件内下载指向 GitHub
- **卡壳**: GitHub Releases 上的模型文件还未上传（需要 VPN 下载后手动上传）

### 🔴 阻塞：GPU PyTorch CUDA 下载 (P0)
- **问题**: CUDA 版 torch wheel (~2.5GB) 只能从国际 CDN 下载
- 清华 pip 镜像只缓存 CPU 版 torch
- GitHub Releases 单文件 2GB 上限，torch wheel 超了

### 🟡 已修复的 Bug
- Whisper 注意力头数计算 → `n_state // 64`
- 重启逻辑 → `done(42)` 返回码 + `_do_restart()`
- 配置保存冲突 → 直接 `config_manager.save()`
- 文件名编码 → `stdout.reconfigure(utf-8)`
- sync CLI dataclass → 属性访问替代 `.get()`
- Provider 自动回退 → 严格模式
