# 在线歌词匹配与 AI 校准指南

Echovault 的“歌词与标签”工作区分为“在线歌词与封面、本地识别编辑、歌词核对”
三个页签。“在线歌词与封面”默认排在第一位，使用 LRCLIB 查找候选歌词，使用
MusicBrainz 与 Cover Art Archive 搜索专辑封面，并提供常用音频标签编辑。
歌词写入只修改 LRC；封面和文字标签写入不会重新编码音轨。

## 1. 从界面使用

1. 切换到排在最前的“在线歌词与封面”，先用“来源”选择全部素材、音乐库或视频库；素材下拉框会以 `[音乐]` / `[视频]` 区分两个库。
2. 软件读取媒体标签；标签不存在时，尝试从 `歌手 - 歌名.ext` 文件名推断。搜索歌名、歌手和专辑仍可手工修正。
3. 点击“一键搜索歌词与封面”。歌词与封面并行请求；左侧歌词表会自动选中最佳候选并在下方显示歌词，右侧同时显示封面缩略图。相同条件的结果缓存 10 分钟，封面缩略图最多四路并发加载。
4. 在线页下方播放器播放当前本地素材，并按时间戳高亮、居中当前歌词。在线歌词可双击编辑，修改会同步到“歌词核对”页。
5. “本地识别编辑”负责生成或更新本地 LRC、编辑和翻译；“歌词核对”提供本地/在线双栏和共用播放器。LRCLIB 候选进入核对区前会自动繁转简，时间戳不变。
6. 在“歌词核对”中根据需要选择四种确定性结果，或继续使用 AI：
   - **采用左侧本地歌词**：保存左侧当前内容。
   - **采用右侧在线歌词**：使用在线候选的完整同步歌词。
   - **合并：左时间轴 + 右文字**：逐行保留本地识别时间轴，换成在线文字。
   - **合并：右时间轴 + 左文字**：采用在线时间轴，换成本地识别文字。
   - **AI 核对并校准左侧歌词**：使用左侧当前编辑的时间轴、右侧当前编辑的参考文字修正左侧，写入后直接刷新左半栏，严格保留左侧时间戳。

## 2. 封面与音频标签

1. 选择音乐后，当前内嵌封面会先显示在在线页右侧；一键搜索会直接在同一区域补充在线候选，也可点“刷新封面”单独重查。
2. 点击在线封面会下载大图并打开写入确认框；也可点“选择本地封面”导入 JPEG/PNG。
3. “音频标签”表单读取当前文件的标题、歌手、专辑、年份和轨道号。“用搜索信息填入”可复制当前搜索条件，确认无误后点“保存音频标签”。
4. 封面与文字标签当前支持 MP3、FLAC、M4A/MP4 音频、OGG、Opus 和 WAV；写入过程不会重新编码音轨。
5. 写入封面后，左侧素材列表会立即在歌曲名称旁刷新小封面。手机传输任务中的音乐
   写入封面后，会作为已修改差异进入待回传列表。

在线封面来自 MusicBrainz 社区维护的发行信息和 Cover Art Archive 图片。
同一歌曲可能存在单曲、专辑、现场版或再版封面，应在写入前查看标题、歌手、
年份和预览。AAC、WMA、APE、WV 当前仍可识别歌词，但暂不写入内嵌封面。

若两侧同步歌词行数不同，交叉合并按各自整段播放进度做单调对齐；纯文本候选则按行序比例映射。写入前应在播放器中人工核对。

歌词区下方的播放器播放**当前选择的本地媒体**。LRCLIB 不托管歌曲音频，因此软件不会播放或下载所谓“在线歌曲文件”。播放、暂停或拖到例如 13 秒时，左右两侧分别按自己的时间轴找到 13 秒对应行，同时加粗、高亮并自动滚动。

在线页、核对页和素材库共用当前选中的素材；批量任务统一提供批量识别、批量翻译和
批量在线匹配。批量在线匹配默认只搜索，只有勾选“自动写入”并再次确认后才会备份并
写入达到阈值的最佳同步歌词。

## 3. 歌词写入、备份与恢复

- 已有 LRC 在任何覆盖或校准前都会备份为 `歌名.lrc.bak`；如果该文件已存在，继续使用 `.bak.1`、`.bak.2`。
- 写入使用同目录临时文件并原子替换；校验或 AI 返回失败时不会改写原文件，也不会留下无效备份。
- “采用右侧”及“右时间轴 + 左文字”采用在线记录自己的时间戳，因此 LRC 时间轴可能变化，但媒体文件始终不变。
- 两种交叉合并都会保留被选作“时间轴一侧”的原始时间戳格式，包括百分秒、毫秒和同一行多个时间戳。
- “AI 核对并校准”要求本地 LRC 已存在。软件严格保留本地的时间戳前缀，包括毫秒精度以及一行多个时间戳；AI 返回行数不一致时拒绝写入。
- 要恢复时，关闭正在编辑该 LRC 的程序，再将对应 `.bak` 复制回 `.lrc`。

## 4. AI 接口

AI 校准复用“设置 → 本地部署 AI”中当前选择的接口：可以是 DeepSeek 在线接口，也可以是 Ollama、LM Studio 或其他 OpenAI 兼容本地服务。请求只包含歌曲元数据、本地歌词文字和在线参考歌词，不包含音频。

返回必须是与本地时间轴行数完全相同的 JSON：

```json
{"corrections": ["校准后的第一行", "校准后的第二行"]}
```

本地接口、鉴权、请求结构和隐私说明见 [AI 接口与本地模型接入指南](AI接口与本地模型接入指南.md)。

## 5. CLI

```powershell
# 搜索是只读操作，结果包含记录 ID 和匹配分数，不打印完整歌词
python main.py lyrics online-search "Never Gonna Give You Up" `
  --artist "Rick Astley" --duration 213 --json

# 下载指定记录的同步歌词；FILE 可为媒体或 LRC
python main.py lyrics online-apply E:\music\song.mp3 --id 3396226 --json

# 以指定在线记录校准现有本地 LRC，保留本地时间戳
python main.py lyrics calibrate E:\music\song.lrc --id 3396226 --json
```

`online-search` 是 AI/CLI 白名单中的只读命令。`online-apply` 和 `calibrate` 会写文件，因此通过 MCP 使用时必须同时满足：服务以 `--allow-writes` 启动，并且本次调用传入 `confirmed=true`。

## 6. 网络、来源与使用边界

- 搜索条件和记录 ID 会发送到 LRCLIB；下载歌词时会读取对应公开记录。无需在 Echovault 中填写 LRCLIB API Key。
- LRCLIB API 当前搜索端点不提供分页，并限制返回 20 条；搜不到时请简化专辑名、调整歌手或歌名后重试。
- 公共歌词可能存在错配、不同版本、时间偏差或权利限制。写入前应人工核对，歌词的使用与分发责任仍由用户承担。
- Echovault 为请求发送可识别的 `User-Agent`，并遵循 LRCLIB 的公开 API 约定。
- 封面搜索条件会发送到 MusicBrainz，图片从 Cover Art Archive 下载；MusicBrainz
  请求遵守其公开 API 的 User-Agent 和速率限制要求。

参考资料：

- [LRCLIB 项目主页与开源仓库入口](https://lrclib.net/)

## 7. 常见问题

**搜索不到结果**：先只保留歌名和歌手；去掉版本、括号、音质等文件名后缀。纯录音或未公开歌曲通常没有匹配。

**“采用右侧”或“右时间轴 + 左文字”不可用**：所选记录只有纯文本歌词，没有同步时间轴。换一条同步候选；纯文本仍可作为“左时间轴 + 右文字”或 AI 校准的文字来源。

**AI 校准拒绝写入**：通常是 AI 没有返回合法 JSON，或返回行数与本地带时间戳歌词不一致。检查当前 AI 接口后重试；原 LRC 不会因此被修改。

**匹配度高但时间不准**：文字相似度不衡量时间轴。优先使用“左时间轴 + 右文字”或“AI 核对并校准”保留本地 ASR 时间；也可播放本地素材，观察两侧实时高亮后再决定。

**搜到的封面不对**：补全专辑名和歌手，避免只用常见歌名；在候选中核对年份。
仍无合适结果时使用“本地封面”。
