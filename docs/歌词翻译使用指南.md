# Echovault 歌词翻译使用指南

Echovault 支持 AI 接口翻译和 Argos 本地离线翻译。两种方式都会生成独立译文 LRC，不覆盖原歌词，并原样保留每一行的时间戳前缀。

## 1. 在详情页翻译

1. 在素材列表选择一首已有 LRC 的歌曲或视频。
2. 在右侧“详情 → 歌词预览与翻译”选择翻译引擎、源语言和目标语言。
3. 点击“翻译当前”。
4. 完成后预览自动切换到译文；用“查看原文/查看译文”来回切换。

译文命名规则：

```text
原文：Song.lrc
中文译文：Song.zh.lrc
英文译文：Song.en.lrc
```

时间戳不会被重新计算。例如 `[00:04.50][00:08.50]World` 翻译后仍保留两个完全相同的时间戳。

## 2. AI 翻译

“AI 翻译”使用“设置 → 本地部署 AI”中当前选中的接口：

- 在线 AI：默认 DeepSeek，需要 Key，会把歌词文字发送到在线服务。
- 本地部署 AI：Ollama、LM Studio 或其他 OpenAI 兼容接口；是否离开本机取决于用户填写的接口地址。

软件按最多 40 行分块，要求 AI 返回等长 JSON 数组。只要返回行数不一致或 JSON 无法解析，本块翻译就会失败且不会写入不完整文件。

AI 接口格式见 [AI 接口与本地模型接入指南](AI接口与本地模型接入指南.md)。

## 3. 本地翻译库

本地离线翻译基于开源 Argos Translate：

1. 打开“设置 → 歌词输出”。
2. 选择源语言和目标语言。
3. 点击“下载所选本地翻译库”。
4. 状态显示“已安装 ✓”后，在详情页把引擎切换为“本地库”。

语言模型按需下载，不随所有安装包预装。模型保存在 Argos 的当前用户数据目录；安装后翻译过程不访问在线 AI。某些语言方向没有直接模型时，需要分别安装经英语中转的语言包；Argos 支持使用已安装模型自动中转，但翻译质量可能下降。

源码环境安装本地运行时：

```powershell
pip install -r requirements-translation.txt
```

## 4. 批量翻译

“批量翻译”处理当前详情列表中所有已有原文 LRC 的素材。开始前会显示文件数量并要求确认。每个文件独立生成目标语言后缀文件；单个文件失败不会删除其他成功结果，结束时会汇总成功和失败数量。

## 5. CLI 与 MCP

翻译单个文件：

```powershell
python main.py lyrics translate E:\Music\Song.lrc --engine local --source en --target-language zh --json
```

批量翻译目录：

```powershell
python main.py lyrics translate E:\Music --engine ai --source en --target-language zh --json
```

省略 `--engine`、`--source` 或 `--target-language` 时使用本机配置。`lyrics translate` 会创建文件，因此在内置 AI 和 MCP 白名单中属于写操作，仍需确认；MCP 还必须以 `--allow-writes` 启动。

## 6. 安全与恢复

- 原始 `.lrc` 不会被覆盖。
- 输出先写入同目录临时文件，刷新并原子替换目标译文，异常时不留下半份译文。
- 在线 AI 只接收歌词文字和翻译要求，不接收音频文件。
- 删除译文只需删除 `.<语言>.lrc`；不会影响原文或音轨。
- 版权和服务条款由素材使用者负责；不要把无权上传的歌词发送到在线服务。
