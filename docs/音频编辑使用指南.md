# 音频编辑使用指南

## 1. 入口和设计原则

先在素材库选择音乐或视频，再打开右侧“音频编辑”。首页参考移动端音频工具的宫格结构，
但每个入口都会展开为完整桌面参数页，不是只显示一个按钮。

所有处理默认遵守三条规则：

1. 原素材保持不变。
2. 新文件默认保存到原素材旁的 `Echovault编辑输出`。
3. 如果原素材属于手机接收任务，生成文件会登记到“待回传”目录。

## 2. 工具说明

| 工具 | 输入 | 主要参数 | 输出 |
| --- | --- | --- | --- |
| 文件管理 | 当前素材 | 无 | 显示时长、采样率、声道、大小和本次结果 |
| 提取音频 | 视频或音频 | 输出格式 | 独立音轨 |
| 音频剪辑 | 单个音频 | 开始、结束、淡入、淡出 | 新剪辑 |
| 录音 | 默认麦克风 | 保存位置 | 16-bit WAV |
| 音频裁剪 | 单个音频 | 开始、结束 | 指定时间范围 |
| 音频拼接 | 多个音频 | 文件顺序 | 首尾连续音频 |
| 音频混合 | 多个音频 | 各轨音量倍率 | 同时播放的混音 |
| 淡入淡出 | 单个音频 | 淡入/淡出秒数 | 平滑首尾 |
| 变速变调 | 单个音频 | 0.25–4 倍速、±12 半音 | 调速调音结果 |
| 音频降噪 | 单个音频 | 噪声底限 | FFT 底噪抑制结果 |
| 音量归一化 | 单个音频 | 目标 LUFS、真峰值 | 统一主观响度 |
| 音频分割 | 单个音频 | 每段秒数 | 连续编号的多个片段 |
| 均衡器 | 单个音频 | 低/中/高频增益 | 三段 EQ 结果 |
| 音频音量 | 单个音频 | 分贝增益 | 放大或减小音量 |
| 音频标记 | 常见音乐格式 | 标题、歌手、专辑、年份、轨道号 | 带新标签的副本 |
| 更多功能 | 单个音频 | 格式、声道、采样率、倒放 | 转换或反向音频 |

## 3. 操作流程

1. 选择工具。
2. 确认输入文件；拼接和混音每行填写一个文件，顺序即处理顺序。
3. 修改详细参数和输出文件。
4. 可先点击“试听输入”，处理后点击“试听输出”。
5. 点击蓝色执行按钮，等待底部进度结束。
6. 在“文件管理”双击本次结果，或打开输出目录。

## 4. 手机回传

手机 LocalSend 传入的文件会被归入独立传输任务。使用音频编辑生成结果后，软件根据源文件
找到该任务，将结果暂存到任务对应的待回传目录，然后自动刷新“手机传输”页面。

回到“手机传输”后可查看新增文件、勾选需要的结果并发送。发送成功的待回传副本会移入
已发送缓存；`Echovault编辑输出` 中的正式结果不会因此删除。

## 5. 实现与兼容性

- 音频处理使用随程序打包的 FFmpeg。
- WAV/MP3/FLAC/M4A/OGG/Opus 等常见输出会选择对应编码器。
- 标签使用 Mutagen 写入，不重新编码音频流。
- 录音使用 Qt Multimedia 和 Windows 当前默认输入设备。
- 降噪采用 FFmpeg `afftdn`，归一化采用 `loudnorm`，分割采用 segment muxer。

具体格式能否写入取决于源文件容器和本机/打包 FFmpeg 对应编码器。遇到不支持的素材时，
错误信息会显示 FFmpeg 返回的最终原因，原文件不会被改动。
