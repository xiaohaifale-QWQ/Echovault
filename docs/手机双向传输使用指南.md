# Echovault 手机双向传输使用指南

## 工作流程

手机传输页按一次 LocalSend 发送建立一个任务：

```text
手机发送素材
  → Echovault 记录接收时文件清单
  → 使用现有功能处理素材
  → 自动扫描新增和修改文件
  → 查看并勾选结果
  → 发送回手机
```

## 从手机接收

1. 打开右侧“手机传输”。
2. “打开目录”用于查看当前接收地址；“选择目录”用于更换并保存接收地址。
3. 选择接收目录并点击“开启接收”。
4. 确保电脑和手机位于同一局域网，手机打开 LocalSend。
5. 在手机上选择 `Echovault` 并发送文件。

每一批文件保存到独立子目录，并建立包含发送设备、时间、文件大小和
SHA-256 的传输任务。大文件以流式方式写入，不会一次性放进内存。
接收端会在文件完整落盘后立即向手机返回明确的 HTTP/1.1 成功响应，再建立任务
清单，避免手机显示失败但电脑实际已经收到文件。

## 处理和刷新结果

收到文件后，可以继续使用：

- 歌词识别和在线歌词。
- AI 或 Argos 歌词翻译。
- 人声与伴奏分离、降噪和去混响。
- 视频时间轴与汇总。

这些功能生成输出时会登记到对应传输任务。回到手机传输页点击“刷新结果”，
页面会比较接收时基线和当前文件：

- 新生成、已修改：默认勾选。
- 未变化的原始媒体：默认不勾选。
- 已删除：只提示。
- `.bak`、`.part`、`.tmp` 和缓存：不进入默认回传列表。

双击 LRC、TXT、JSON 或 CSV 可以查看接收时版本和当前版本的文本差异。
媒体文件会显示大小、时长、采样率等信息，也可以交给系统播放器打开。

登记的处理结果会在素材目录的 `Echovault输出\待回传` 建立专用回传副本。
同一 NTFS 磁盘优先建立硬链接，不会为大型人声/伴奏 WAV 再占一份实际数据；
无法建立硬链接时才复制文件。

## 发送回手机

1. 手机保持 LocalSend 打开。
2. 点击“刷新设备”。
3. 在设备下拉框选择手机。
4. 检查勾选文件和总大小。
5. 点击“发送选中的文件到手机”。

Echovault 使用 LocalSend Protocol v2.1 的标准上传接口，并校验发现设备的
HTTPS 证书指纹。手机拒绝、正在处理其他传输、网络中断或用户取消时会显示
明确错误；成功和跳过的文件会写入任务回传历史。

发送成功后，待回传副本自动移入：

```text
~/.music-lyrics-sync/cache/sent-transfer/
```

它不会继续出现在传输页的差异列表中。源素材和正式处理结果仍保留原位。
“设置 → 缓存”会显示语音缓存、已发送文件缓存各自的文件数量和逻辑大小；
清理缓存不会删除尚未发送的待回传文件。

## 手机保存位置

标准 LocalSend 只负责把文件交给手机。文件在手机上的最终目录由手机 LocalSend
设置和 Android/iOS 权限决定，Echovault 不能直接指定或覆盖手机任意原目录。

如需自动放回手机原文件夹，需要独立的手机伴侣端和系统文件授权，不属于当前
LocalSend 桌面协议范围。

## 高级文件夹同步

页面底部保留“高级文件夹同步”，用于两个电脑可访问目录、移动硬盘或映射网络盘。
其中包含 A/B 单向、双向和镜像同步。它不代表手机文件系统，也不用于普通回传流程。
