# 人声与伴奏分离使用指南

## 功能位置

主窗口右侧打开“人声分离”。页面分为上下两部分：

- 上方“处理设置”选择输出内容、输出目录、分离模型和 CPU/CUDA 运行设备；
  待处理素材直接使用左侧当前选中项，不重复显示歌曲下拉框。
- 下方“音量调节台”显示伴奏、人声两条波形，共用播放位置，但音量可以分别调整。

音频和视频都可以作为输入。视频只读取其中的音轨，不修改原视频。

## 第一次使用

1. 源码环境先执行 `pip install -r requirements-local.txt`；发行版已包含运行时。
2. 点击顶栏“模型库”，在“音频分离模型”白色卡片中选择并下载模型。
3. 下载一个模型。下载过程支持续传、取消和 SHA-256 前缀校验。
4. 在左侧选择素材，回到处理页选择输出内容和目录，再点击“开始处理”。

模型文件来自 Demucs 官方模型地址，保存在当前用户的 Torch Hub 缓存目录，一般为
`~/.cache/torch/hub/checkpoints`。删除软件不会自动删除模型缓存。

## 模型选择

| 模型 | 特点 | 建议场景 |
| --- | --- | --- |
| HTDemucs（推荐） | 质量、速度和下载体积较均衡 | 大多数歌曲、第一次使用 |
| HTDemucs Fine-tuned | 四个精调模型组合，细节更好但约慢四倍 | 最终导出、难分离素材 |
| MDX Extra Quantized | 量化模型，较轻、预览较快 | 低配电脑或快速试听 |

官方 Demucs 文档说明 `htdemucs` 是默认模型，`htdemucs_ft` 是精调版本；项目与 API：
<https://github.com/facebookresearch/demucs>、
<https://github.com/facebookresearch/demucs/blob/main/docs/api.md>。

## 输出文件

“输出内容”可选择“人声 + 伴奏”“仅人声”或“仅伴奏”。处理完成后输出目录按选择生成：

- `原文件名_vocals.wav`：人声音轨；
- `原文件名_accompaniment.wav`：原始混音减去人声后的伴奏音轨。

两条结果使用无损 WAV，时间长度和采样率一致。软件不会覆盖输入素材。

## 试听和导出

处理完成后：

1. 在“输出”下拉框选择实际使用的 Windows 扬声器，再点击“试听”。两条音轨会从同一位置同步播放。
2. 拖动进度条可同时跳转；红色播放线会在两条波形上同步移动。
3. 分别调整“伴奏音量”和“人声音量”。试听音量立即生效。
4. 同时输出两条音轨时，可点击放大的“保存调音结果”，选择新的 WAV 文件。软件通过内置 ffmpeg 合成，不修改两条分离结果；只输出单条音轨时无需再次保存。

100% 表示原分离音轨增益，0% 表示静音。两条均设为 100% 时，峰值可能接近或超过原始
幅度；如出现削波，可同时降低两个滑块后重新导出。

## 用于本地歌词识别

“设置 → 语音识别”中的“启用 Demucs 人声分离”现在会真正参与本地识别：

1. 使用 HTDemucs 生成临时人声音轨；
2. Whisper 只识别人声音轨；
3. LRC 仍按原素材命名和时间轴保存；
4. 识别完成或失败后删除临时分离文件。

使用前必须在模型库安装 HTDemucs。该选项适合伴奏很响、和声复杂或现场录音；清晰的纯人声
素材无需开启，因为分离本身会增加处理时间。

## 硬件与隐私

- CPU：所有电脑可用，但长歌曲可能需要数分钟。
- NVIDIA CUDA：顶栏“模型库”的两张卡片分别显示文字识别与音频分离 GPU 状态。
  两类模型文件都可由 CPU/GPU 共用，不需要重复下载；Demucs 只有在主程序 CUDA
  PyTorch 就绪时才会显示 CUDA，外置 ASR Worker 不能直接替代它。
- 所有分离均在本机完成，输入音频不会发送到网络；只有下载模型时访问官方模型服务器。

## 常见问题

### 提示“模型尚未下载”

打开顶栏模型库安装当前下拉框选中的模型；识别前分离也使用该默认模型。

### 模型下载校验失败

软件会删除损坏的临时文件。检查网络、代理和磁盘空间后重新下载。

### CUDA 处理失败或显存不足

切换到 CPU 后重试；关闭其他占用显存的程序。精调模型比推荐模型占用更多时间和资源。

### 两条音轨听起来仍有串音

源混音中的混响、合唱、立体声效果会使任何分离模型都产生残留。可换用
HTDemucs Fine-tuned；“降噪”和“混响移除”只有在对应模型正式接入后才会开放，当前界面
明确禁用，不会假装处理。
